# Security Policy

LawLink 面向律师业务数据，默认把案件事实、客户信息、附件、财务记录和审计日志视为敏感数据。

## 支持范围

当前安全修复优先覆盖默认分支和最新公开版本。早期版本不承诺长期安全维护。

## 报告安全问题

请不要在公开 issue 中披露可利用的漏洞、密钥、真实案件数据或复现细节。

推荐方式：

1. 使用 GitHub 的私密安全报告功能（Security Advisories / Report a vulnerability）。
2. 如果仓库尚未开启该功能，请先创建一个不含攻击细节的普通 issue，标题写 `Security contact request`，等待维护者给出私密沟通方式。

报告时请包含：

- 影响范围。
- 复现条件。
- 可能造成的后果。
- 你建议的修复方向。

请不要包含真实客户资料、真实案件材料、真实密钥或未打码的个人信息。

## 部署方责任

自部署使用者需要自行负责：

- 更换 `.env` 中的所有默认值。
- 妥善保管 `NEXTAUTH_SECRET` 和 `STORAGE_ENCRYPTION_KEY`。
- 配置 HTTPS、反向代理、防火墙和数据库访问控制。
- 定期备份数据库与 `storage/`，并单独备份文件加密主密钥。
- 控制管理员账号和服务器访问权限。

如果 `STORAGE_ENCRYPTION_KEY` 丢失，已加密文件无法恢复。
